# RFC-001 — Product Direction Correction

> STATUS: ACCEPTED

---

# Original Direction

Original project direction:

Generic AI workspace platform.

Focus areas included:
- agent orchestration
- briefs
- creative workflows
- divergence/convergence systems
- multi-agent interaction surfaces

---

# Corrected Direction

Project direction evolved into:

Foundry — AI-native operational runtime orchestration system.

Core focus shifted toward:
- operational continuity
- runtime orchestration
- workspace isolation
- checkpoint/recovery systems
- deployment/runtime governance
- AI-assisted software/runtime engineering

---

# Rationale

The generic AI workspace direction created:
- vague primitives
- weak operational identity
- excessive abstraction
- insufficient differentiation

The Foundry direction provides:
- stronger operational philosophy
- clearer architectural boundaries
- safer AI mutation strategy
- runtime-centric system identity
- scalable isolation model

---

# Foundational Principle

Foundry Core remains stable and protected.

Constructs/workspaces become isolated mutable runtimes.

---

# Implications

This correction affects:
- UX language
- thread surfaces
- entry primitives
- architecture planning
- deployment strategy
- recovery systems
- future AI orchestration

---

# Status

Accepted and active.

All future architecture should align with:
Foundry as operational runtime infrastructure.
